# IPv4 vs IPv6：網際網路協議的演進

## IPv4 簡介

IPv4（Internet Protocol version 4）是目前最廣泛使用的網際網路協議版本。它使用 32 位元地址，格式為四個由點分隔的 8 位元組（例如：192.168.1.1）。

### IPv4 面臨的問題

1. **地址耗盡**：32 位元地址空間僅能提供約 43 億個唯一地址，遠遠不足以滿足當前互聯網設備的需求。
2. **安全性問題**：原生不支持加密和身份驗證。
3. **配置複雜**：需要手動配置或使用 DHCP。
4. **服務質量（QoS）支持有限**：難以有效管理不同類型的網路流量。

### IPv4 中的通信模式

1. **單播（Unicast）**：一對一通信，數據包從源地址發送到單一目標地址。
2. **廣播（Broadcast）**：一對所有通信，數據包從源地址發送到網段內的所有設備。
3. **多播（Multicast）**：一對多通信，數據包從源地址發送到一組特定的目標地址。

## IPv6 簡介

IPv6（Internet Protocol version 6）是為了解決 IPv4 問題而開發的下一代網際網路協議。

### EUI-64

EUI-64（Extended Unique Identifier）是一種用於生成 IPv6 接口標識符的方法：

- 將 48 位 MAC 地址轉換為 64 位接口標識符。
- 在 MAC 地址的中間插入 FFFE，並將第 7 位取反。
- 用於生成 IPv6 地址的後 64 位。

## 結論

IPv6 不僅解決了地址耗盡問題，還通過多樣化的地址類型和靈活的配置方式，為網絡設計和管理提供了更多可能性。從 SLAAC 到 DHCPv6，IPv6 為不同場景提供了適合的解決方案，大大提高了網絡的靈活性和效率。隨著 IPv6 的廣泛部署，這些優勢將為互聯網的未來發展奠定堅實的基礎。
